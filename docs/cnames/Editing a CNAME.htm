Editing a CNAME

  1. In the left pane, click CNAMEs.
  2. Click the CNAME you want to edit.

    OR

    Click the Actions drop-down list for the CNAME you want to edit, and then select Edit.
  3. At the top right of the page, click the Edit button.

(click to enlarge)

The CNAME form that appears allows you to create client region rules, edit client region rules, delete client region rules, and change advanced settings.

(click to enlarge)

  1. To create one or more client region rules, perform the following steps.

Note: If you do not create a client region rule, or if you leave the Client Region field empty, a rule covering ALL regions is created automatically.

  1. At the top right of the page, click the Create Client Region Rule button.

(click to enlarge)

  1. Complete all the fields, and then click the Create Client Region Rule button.
  2. To add more client region rules, repeat this step for each additional rule.

 
Fields Description
Client Region Select a region for this rule. If you leave this field empty, the ALL region is selected for the CNAME automatically.
Client ISP Select a client ISP.
Action type Select the type of action to be performed. Choices are:
  • Deliver = specifies the server groups to deliver your traffic. Only one deliver action is allowed for each client region. All servers in the selected server group will be used as candidates in the load balancing algorithm.
  • Redirect = directs traffic requests to the target specified in the Redirect target field (see below). This can be your origin site or another CDN provider. There can be multiple redirect actions for each client region.
  • Reject = directs traffic requests to web servers that always respond with a 403 - forbidden error message. Only one reject action is allowed for each client region.
Server Group If Action Type is set to Deliver, select one or more server group options:
  • Standard = standard server group.
  • Premium = includes the standard server group.
  • Premium+ = includes the standard and premium server groups.
  • Ultra = includes the standard, premium, and premium+ server groups.
Redirect target If Action Type is set to Redirect, specify an IP address or hostname to which CDN360 will redirect your traffic.
Weight Adjust how this rule behaves relative to other rules for the same client region.
  1. To edit an existing client region rule:
  1. Click the Actions drop-down list next to the rule you want to edit, and then select Edit.

  2. Click the Update Client Region Rule button.

  1. To delete a client region rule:
  1. Click the Actions drop-down list next to the rule you want to delete, and then select Delete.

(click to enlarge)

  1. When prompted to confirm the deletion, click OK to delete the client region rule.
  1. To change the Has Beian setting, expand Advanced Settings and make your change.

(click to enlarge)

 
Fields Description
Has Beian

Select whether content will be served from PoPs inside or outside China. Choices are:

  • No = content is served to website visitors in China from PoPs located outside China. (default)
  • Yes = content is served to website visitors in China from PoPs located in China.
  1. Click Update.
  2. When a message confirms that the CNAME was updated click OK.
  1. Click Close to return to the main page.
